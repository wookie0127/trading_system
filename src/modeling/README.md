# Yahoo Finance 데이터 수집 제한 사항 (Data Limitations)

본 프로젝트에서 시장 데이터를 수집할 때 사용하는 `yfinance` 라이브러리는 데이터의 단위(Interval)에 따라 조회 가능한 과거 기간(Lookback) 및 한 번의 요청으로 가져올 수 있는 기간에 제한이 있습니다.

## 1. 단위별 제한 사항 요약

| 단위 (Interval) | 최대 요청 가능 기간 (per Request) | 최대 과거 조회 범위 (Lookback) | 비고 |
| :--- | :--- | :--- | :--- |
| **1m** | **7일 (7d)** | **최근 30일 이내** | 7일 초과 요청 시 에러 발생 |
| **2m, 5m, 15m** | 60일 (60d) | 최근 60일 이내 | 3개월치 일괄 수집 불가 |
| **30m, 90m** | 60일 (60d) | 최근 60일 이내 | |
| **60m, 1h** | 730일 (730d) | 최근 730일 이내 | |
| **1d, 1wk, 1mo** | 제한 없음 | 상장 이후 전체 | |

## 2. 주의사항

- **1분봉(1m) 30일치 수집**: 1분봉 데이터를 30일치를 채우고 싶다면, 한 번에 요청하지 말고 **7일 단위로 청크(Chunk)**를 나누어 여러 번 호출해야 합니다. (`src/backfill/market_bulk_backfill.py` 참고)
- **3개월 데이터 확보**: 인트라데이 데이터(1m~15m)의 경우 3개월(90일) 데이터를 Yahoo Finance에서 직접 한 번에 가져오는 것은 불가능합니다. 이를 위해서는 주기적인 수집기 실행을 통해 로컬에 데이터를 축적해야 합니다.

## 3. 검증 스크립트
`src/modeling/check_data_range.py` 스크립트를 사용하여 요청하려는 파라미터가 유효한지 확인할 수 있습니다. 유효하지 않은 기간을 요청할 경우 `ValueError`를 발생시킵니다.
