# Data Leakage Review

## 1. 개요
시스템 트레이딩 백테스트 및 Replay 시 발생 가능한 데이터 누수(Look-ahead Bias / Data Leakage) 요소를 분석하고 방지책을 정의합니다.

## 2. 주요 누수 위험 요소 및 방지책
1. **4시간봉 미완성 캔들 사용**
   - *위험*: 캔들이 종가 확정되기 전(예: 03:59:59) 지표를 계산하면 미래 정보 반영.
   - *방지*: `timestamp`가 완전 종가 확정(e.g., `close_time <= current_snapshot_time`)된 캔들만 수집.
2. **뉴스 및 외생 변수 시점 불일치**
   - *위험*: 판단 시점 이후에 수집된 뉴스/거시 지표가 과거 스냅샷에 포함됨.
   - *방지*: 뉴스 및 거시 지표에 `published_at` 타임스탬프를 부여하고, 스냅샷 생성 시점(`effective_at`) 이전 데이터만 필터링.
3. **지표 계산 시 전체 시리즈 룩어헤드**
   - *위험*: 전체 데이터프레임에 대해 센터링 이동평균이나 미확정 윈도우 계산 사용.
   - *방지*: 순차적 Point-in-time 슬라이싱을 통해서만 지표 계산.
